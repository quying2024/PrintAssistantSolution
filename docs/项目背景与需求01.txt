我有一个关于辅助办公流的项目需求（项目名称：PrintAssistant）：它是一个托盘程序，功能是监控指定文件夹，如果有新添加的文件，将其发送到物理打印机打印出来。

项目细节需求：

1、本项目不是一个全自动的工作流，而是一个人工辅助工具。

2、监控文件夹由用户指定（默认是桌面上的“PrintJobs”文件夹）

3、对于连续添加的新文件，应视为一组，统一打印，统一计算页码，统一计算为一个任务。一个任务确定后，在完成过程中又添加的新文件既新任务，不得遗漏，多个任务，顺序排队处理。

4、对于不支持的文件类型应单独处理（移至指定的文件夹），并提示用户自行处理。

5、待打印文件类型包括doc/docx/xlsx/jpg/png/bmp/pdf等格式，需要统一转换为pdf格式再打印。

6、打印前需要弹窗显示系统已安装的打印机，供用户选择，要有打印份数选项（默认1份），要允许用户在配置菜单中设定不会向用户显示的打印机，弹窗需置顶显示。

7、一个任务结束完成后，本任务的文件均移至以时间戳命名的子目录中。

8、要允许用户设置日志文件的存储位置（默认C:\Users\用户名\AppData），要有定期清理日志文件的选项（自动删除超过1周的日志文件）。

9、鼠标在托盘图标上悬停应显示最近的5个任务列表（包含任务序号/时间/总页数等），最新的任务在最上边，用户用鼠标双击某个任务项时，应弹出存放该任务文件的文件夹。

10、鼠标在托盘图标上右键单击应弹出配置菜单，包含监控文件夹/非显示打印机/日志文件目录等由用户自行设置的项目

11、要有完善的容错机制及错误处理机制。
12、由于项目主要在中文环境下运行，请确保所有项目文档与注释统一编码（UTF-8 BOM)，防止乱码产生。

项目技术栈：

• 目标框架: .NET 8.0 (Windows)

•	开发语言: C# 12.0

•	平台支持: Windows x64 专用

•	Windows Forms: 桌面UI框架，用于托盘图标和配置界面

•   PDF处理技术栈：Syncfusion.DocIO，我已申请到Syncfusion.DocIO的社区免费密钥，请充分利用该商业库的功能

特别要求：

项目架构必须保证满足单元测试要求，每一个模块/类都要有注释说明输入输出类型及要求
